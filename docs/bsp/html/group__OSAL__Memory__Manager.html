◆ R_OSAL_MmngrAlloc()

e_osal_return_t R_OSAL_MmngrAlloc ( osal_memory_manager_handle_t  hndl_mmngr,
size_t  size,
size_t  alignment,
osal_memory_buffer_handle_t *const  p_buffer_obj 
)

Allocate memory from the memory manager hndl_mngr with the given size and alignment and default memory attributes of the memory manager.

Parameters
[in]hndl_mmngrHandle to the Memory Manager to allocate the memory from.
[in]sizeRequested size of the memory: range n*minimumCachelineSize ... maxBufferSize where n = 1,2,3...
[in]alignmentAlignment of the memory [range: 2^6, 2^7, ..., 2^n with n limited by the system or MMNGR ]
[in,out]p_buffer_objHandle to the allocate buffer object
Returns
e_osal_return_t
Return values
OSAL_RETURN_OKAll arguments are valid, and the memory was allocated successfully.
OSAL_RETURN_PARInvalid argument.
  • p_buffer_obj is NULL.
OSAL_RETURN_MEMMemory manager is not capable of allocating requested memory with the given parameters. E.g. run out of memory
OSAL_RETURN_DEVUnderlying memory manager cannot be accessed.
OSAL_RETURN_TIMEOperation took too long.
OSAL_RETURN_CONFThe given size and alignment values to not match with the memory managers requirements for allocation memory. please see also st_osal_mmngr_config_t.
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_HANDLEInvalid handle.
  • hndl_mmngr is not acquired by R_OSAL_MmngrOpen().
  • hndl_mmngr was already closed by R_OSAL_MmngrClose().
OSAL_RETURN_STATE
  1. Not initialized. (Before calling R_OSAL_Initialize().)
  2. The Memory Manager is not opened. (Before calling R_OSAL_MmngrOpen().)
OSAL_RETURN_PROHIBITEDThe maximum amount of allowed allocations is reached.
OSAL_RETURN_HANDLE_NOT_SUPPORTEDThe provided mmngr does not support this operation.
Note
This function will always use the default memory attributes of the memory manager. OSAL Memory Manager only supports one set of attributes. If you want to allocate memory with different attributes, please use the Specific Memory Manager function (e.g. future HWA Memory Manager).
Similar to standard allocations the actually allocated memory may be larger than the requested size to fit alignments, avoid fragmentations and underflow/overflow detections. The reported allocated size of the buffer.
See also
  • R_OSAL_MmngrGetSize() will always match the user requested size.
  • st_osal_mmngr_config_t

◆ R_OSAL_MmngrCheckBuffer()

e_osal_return_t R_OSAL_MmngrCheckBuffer ( osal_memory_buffer_handle_t  memory_buffer_obj_hndl)

Triggers a checks the current buffer object for integrity. E.g. by checking for underflow or overflow. Behavior may differ between memory buffer objects and their manager implementation.

Example 1:
The buffer is CPU read/write accessible and mmngr has underflow/overflow detection enabled:
This will function will trigger check of any unexpected modifications of the underflow/overflow memory regions and will result in issuing a monitor event call. (Please see e_osal_mmngr_event_t)

Example 2:
The buffer is not CPU read/write accessible but IOMMU/IPMMU Guard pages are used to detect underflow/overflow of memory regions by HW. This will check if regions assigned/neighbored to this buffer object are marked by the mmngr to be modified. This situation would be automatically be reported by the MMNGR by calling the matching monitor. This function would trigger this event again.

Parameters
[in]memory_buffer_obj_hndlHandle of the buffer object.
Returns
e_osal_return_t
Return values
OSAL_RETURN_OKAll arguments are valid, and the current buffer object for integrity is ok. (no overflow/underflow)
OSAL_RETURN_MEMThe buffer integrity is not ok (overflow/underflow)
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_HANDLEInvalid handle.
  • memory_buffer_obj_hndl is NULL.
  • memory_buffer_obj_hndl is not acquired by R_OSAL_MmngrAlloc().
  • handle->p_ptr_self is invalid. handle might be corrupted or it is not acquired by OSAL memory manager.
OSAL_RETURN_UNSUPPORTED_OPERATIONThe provider buffer object does not support this operation.
Note
This operation must be supported by the buffer object and mmngr.
See also
e_osal_mmngr_event_t

◆ R_OSAL_MmngrFlush()

e_osal_return_t R_OSAL_MmngrFlush ( osal_memory_buffer_handle_t  memory_buffer_obj_hndl,
size_t  offset,
size_t  size 
)

Synchronizes a sub region of the buffer CPU memory to the HW.
Synchronizes CPU memory to the HW by making sure the memory content accessible by the HW is identical with the CPU memory view.

Parameters
[in]memory_buffer_obj_hndlHandle of the buffer object
[in]offsetOffset with the buffer object limits [range: 0, n*minimumCachelineSize ... maxBufferSize] n = 1,2,3,... (e.g. 0, n* 64 Byte)
[in]sizeSize of the region to be synced [range: [buffer_specific, typical 64Bytes]–buffer_size].
Returns
e_osal_return_t
Return values
OSAL_RETURN_OKAll arguments are valid, and the sub region of the buffer CPU memory was synchronized to the HW successfully.
OSAL_RETURN_PARInvalid argument.
  • offset is out of range for the given buffer object.
  • offset is not aligned 64 byte.
  • size is out of range for the given buffer object.
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_HANDLEInvalid handle.
  • memory_buffer_obj_hndl is NULL.
  • memory_buffer_obj_hndl is not acquired by R_OSAL_MmngrAlloc().
  • handle->p_ptr_self is invalid. handle might be corrupted or it is not acquired by OSAL memory manager.
OSAL_RETURN_PROHIBITEDWrite access by CPU is prohibited to the buffer object.
OSAL_RETURN_UNSUPPORTED_OPERATIONThe provider buffer object does not support this operation.
Note
This operation may overwrite memory modified by the HW since the last call to R_OSAL_MmngrFlush(). User must make sure this is avoided or will not have impact.
Provided size must be multiple of the minimal size. Typically multiple of 64Byte (ARM Cache Line Size).
This operation will return OSAL_RETURN_OK also in case no synchronization is required due to CPU memory view and HW memory view is ensured to be same. Example: Memory is uncached.

None.

◆ R_OSAL_MmngrGetConstCpuPtr()

e_osal_return_t R_OSAL_MmngrGetConstCpuPtr ( osal_memory_buffer_handle_t  memory_buffer_obj_hndl,
const void **const  pp_const_cpu_ptr 
)

Get the CPU accessible (read-only) pointer of the buffer object and set to pp_const_cpu_ptr.

Parameters
[in]memory_buffer_obj_hndlHandle of the buffer object
[in,out]pp_const_cpu_ptrpointer to the cpu accessible memory
Returns
e_osal_return_t
Return values
OSAL_RETURN_OKAll arguments are valid, and the CPU read accessible pointer of the buffer object was returned successfully.
OSAL_RETURN_PARInvalid handle.
  • pp_const_cpu_ptr is NULL.
OSAL_RETURN_MEMMapping/copying the buffer content to CPU failed.
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_HANDLEInvalid handle.
  • memory_buffer_obj_hndl is NULL.
  • memory_buffer_obj_hndl is not acquired by R_OSAL_MmngrAlloc().
  • handle->p_ptr_self is invalid. handle might be corrupted or it is not acquired by OSAL memory manager.
OSAL_RETURN_PROHIBITEDRead access by CPU is prohibited to the buffer object.
OSAL_RETURN_UNSUPPORTED_OPERATIONThe provider buffer object does not support this operation.
Note
Be aware that acquiring CPU accessible pointer may not be permitted. Call to this function shall be restricted to driver/middleware implementations if reading memory content is not mandatory.
Be aware that any expected modification done by the HW from the point in time this cpu_ptr is acquired must be synced to CPU by calling R_OSAL_MmngrInvalidate(). If this is not done the modified memory content may not be or only partially updated to the CPU.
This operation may require to reserve memory on first call and may fail with OSAL_RETURN_MEM. See documentation of the Memory Manager that created the buffer object for more details.

)

Synchronizes a sub region of the buffer HW memory to the CPU memory.
Synchronizes HW memory with the CPU by making sure the memory content accessible by the HW is identical with the CPU memory view.

Parameters
[in]memory_buffer_obj_hndlHandle of the buffer object
[in]offsetOffset with the buffer object limits [range: 0, n*minimumCachelineSize ... maxBufferSize] n = 1,2,3, ... (e.g. 0, n* 64 Byte)
[in]sizeSize of the region to be synced [range: [buffer_specific, typical 64Bytes]–buffer_size].
Returns
e_osal_return_t
Return values
OSAL_RETURN_OKAll arguments are valid, and the sub region of the buffer HW memory was synchronized to the CPU memory successfully.
OSAL_RETURN_PARInvalid argument.
  • offset is out of range for the given buffer object.
  • offset is not aligned 64 byte.
  • size is out of range for the given buffer object.
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_HANDLEInvalid handle.
  • memory_buffer_obj_hndl is NULL.
  • memory_buffer_obj_hndl is not acquired by R_OSAL_MmngrAlloc().
  • handle->p_ptr_self is invalid. handle might be corrupted or it is not acquired by OSAL memory manager.
OSAL_RETURN_PROHIBITEDWrite access by CPU is prohibited to the buffer object.
OSAL_RETURN_UNSUPPORTED_OPERATIONThe provider buffer object does not support this operation.
Note
This operation may overwrite memory modified by the CPU since the last call to R_OSAL_MmngrInvalidate(). User must make sure this is avoided or will not have impact.
Provided size must be multiple of the minimal size. Typically multiple of 64Byte (ARM Cache Line Size).
This operation will return OSAL_RETURN_OK also in case no synchronization is required due to CPU memory view and HW memory view is ensured to be same. Example: Memory is uncached.

◆ R_OSAL_MmngrOpen()

e_osal_return_t R_OSAL_MmngrOpen ( const st_osal_mmngr_config_t *const  p_config,
osal_memory_manager_handle_t *const  p_hndl_mmngr 
)

Factory method to create the OSAL Memory Manager Instance with the given configuration. To change configuration, the manager must be closed and re-opened. Closing and re-open the OSAL Memory Manager will potentially result in allocation/de-allocation of bookkeeping memory resources and should be limited to initialization phase in restricted environments.

Parameters
[in]p_configConfiguration to instantiate the OSAL Memory Manager with
[in,out]p_hndl_mmngrHandle to the OSAL MMNGR instance
Returns
e_osal_return_t
Return values
OSAL_RETURN_OKAll arguments are valid, and the OSAL Memory Manager was created with the given configuration successfully.
OSAL_RETURN_PARInvalid argument.
  • p_config is NULL.
  • p_hndl_mmngr is NULL.
OSAL_RETURN_MEMInternal Bookkeeping memory could not be allocated.
OSAL_RETURN_DEVOS specific memory resource could not be opened/accessed.
OSAL_RETURN_TIMEOperation took too long.
OSAL_RETURN_CONFRequested Configuration is invalid. please see also st_osal_mmngr_config_t.
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_STATE
  1. The OSAL Memory Manager is already opened.
  2. Not initialized. (Before calling R_OSAL_Initialize().)
OSAL_RETURN_CONF_UNSUPPORTEDRequested allocate mode is not supported. please see also e_osal_mmngr_allocator_mode_t.
Note
In case of calling this API with a config initialized with R_OSAL_MmgrInitializeMmngrConfigSt(), OSAL_RETURN_CONF is returned. Before calling this API, the config value should be initialized with valid value.
OSAL Memory Manager will always allocate memory that is Cached, CPU read/write accessible. IOMMU/IPMMU configuration is not known by OSAL Memory Manager and it is assumed HW has read/write access. IOMMU/IPMMU configuration that changes Virtual address spaces of the HW shall implemented as Specific Memory Manager that is create Buffer objects that can consider IOMMU/IPMMU configurations. This user Memory Manager shall take ownership of the OSAL Memory Manager to prevent other SW from accessing the OSAL MMNGR instance. OSAL Memory Manager cannot be opened twice!

e_osal_return_t R_OSAL_MmngrRegisterMonitor ( osal_memory_manager_handle_t  hndl_mmngr,
e_osal_mmngr_event_t  monitored_event,
fp_osal_memory_mngr_cb_t  monitor_handler,
void *  p_user_arg 
)

Registers a monitor function to a mmngr for monitoring a specific event of the MMNGR This functions registers a monitor/observer to a memory manager for monitoring events (e_osal_mmngr_event_t). Use this monitor functionality to monitoring allocation/deallocation or error events. In safety critical system use this to connect a manager that can handle/detect systematic faults, under- or over-flowing of memory blocks according to ISO26262.

Parameters
[in]hndl_mmngrHandle to the memory manager
[in]monitored_eventEvent that shall be monitored by the provided monitor function
[in]monitor_handlerfunction to be called by the mmngr for the monitored_event
[in]p_user_argUser argument that will be passed without modification/accessed to the monitor handler function.
Returns
e_osal_return_t
Return values
OSAL_RETURN_OKAll arguments are valid, and the monitor function was registered successfully.
OSAL_RETURN_PARInvalid argument.
  • monitor_handler is NULL.
  • monitored_event is out of range. please see also e_osal_mmngr_event_t.
OSAL_RETURN_MEMInternal bookkeeping memory allocation failed.
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_HANDLEInvalid handle.
  • hndl_mmngr is not acquired by R_OSAL_MmngrOpen().
  • handle->p_ptr_self is invalid. handle might be corrupted or it is not acquired by OSAL memory manager.
  • hndl_mmngr was already closed by R_OSAL_MmngrClose().
OSAL_RETURN_STATE
  1. Not initialized. (Before calling R_OSAL_Initialize().)
  2. The Memory Manager is not opened. (Before calling R_OSAL_MmngrOpen().)
OSAL_RETURN_PROHIBITEDThe maximum amount of allowed registered monitor handlers is reached.
OSAL_RETURN_HANDLE_NOT_SUPPORTEDThe provided mmngr does not support this operation.
Note
Any monitoring function shall meet similar requirements as an interrupt handler: Minimal Execution times, non-blocking.
Reason: They are called synchronously from the memory manager and the calling thread and have impact on timing.
User must make sure calling threads that may issue an event have enough stack available for the registered monitor function and the required stack of the mmngr.
The monitor functions may also be used for fault injection during testing. Example: artificially introduce delays.
Same monitor function may be registered for different events but shall not be registered twice for the same event!
