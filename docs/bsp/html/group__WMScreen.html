◆ R_WM_ScreenPropertySet()

r_wm_Error_t R_WM_ScreenPropertySet ( r_wm_Dev_t  WmUnit,
r_wm_ScreenProp_t  Prop,
void *  Data 
)

Initialise the given property of the WM device screen with the given value.

This function sends a message via the OS communication channel to the WM driver. The property must be one of the define one, see r_wm_ScreenProp_t Depending on the property to be set, the Data passed shall be of a specific type:

  • WM_SCREEN_PROP_FMT : pointer to the data for r_wm_ColorFmt_t
  • WM_SCREEN_PROP_BKGCOLOR : pointer to the data for uint32_t which contains ARGB8888; Alpha value shall be ignored
  • WM_SCREEN_PROP_TIMING : pointer to the data for r_wm_DispTiming_t When called twice, the return value will be R_WM_ERR_UNIT_ALREADY_INIT
Parameters
[in]WmUnit- WM Device, range R_WM_DEV_VOUT0 - R_WM_DEV_VOUT3. See r_wm_Dev_t
[in]Prop- Property to be configured
[in]Data- Property Data. Can be a pointer to an integer or a structure depending on the property to be set
Return values
WMerror r_wm_Error_t

◆ R_WM_ScreenSurfacesFlush()

r_wm_Error_t R_WM_ScreenSurfacesFlush ( r_wm_Dev_t  WmUnit,
uint32_t  VSync 
)

Flush the different change requests from the ring buffer

All the surfaces linked to this WM Unit will have their change requests processed. The function can work synchronously and wait for the VBlank to make sure the changes are applied or asynchronously and return immediately. In that case user must sync the application with the VBlank.

Parameters
[in]WmUnit- WM Device, range R_WM_DEV_VOUT0 - R_WM_DEV_VOUT3. See r_wm_Dev_t
[in]VSync- 0, non-blocking, does not wait for VBlank. 1, wait for Vblank before returning
Return values
WMerror r_wm_Error_t
