◆ R_OSAL_IoBlockRead16()

e_osal_return_t R_OSAL_IoBlockRead16 ( osal_device_handle_t  handle,
uintptr_t  offset,
uint16_t *  p_data,
size_t  size 
)

Read a group of 16bit registers from the device and set the read value to p_data.

Parameters
[in]handleTo set the handle acquired at R_OSAL_IoDeviceOpen().
[in]offsetTo set the value of offset from the base address of handle by byte.
[in,out]p_dataTo set the address of uint16_t array. The OSAL sets the register data.
[in]sizeTo set the value of register data size by byte.
Returns
e_osal_return_t
Return values
OSAL_RETURN_OKAll arguments are valid, and successfully read some values from register by 16bit alignment.
OSAL_RETURN_PARInvalid argument.
  • p_data is NULL.
  • offset + sizeof(uint16_t) is more than register address range.
OSAL_RETURN_DEVHW fault detected.
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_IOInput / Output operation failed.
OSAL_RETURN_HANDLEInvalid handle.
  • handle is not acquired by R_OSAL_IoDeviceOpen().
  • handle was already closed by R_OSAL_IoDeviceClose().
OSAL_RETURN_STATENot initialized. (Before calling R_OSAL_Initialize().)
Note
This function provides only access to registers under the assumption that access size is 16bit.
When accessing the same register from different threads or interrupts, ensure that there is no data race on the user side.
In case of any argument other than size is valid and size = 0, this API exits without doing anything and returns OSAL_RETURN_OK. (= Read of 0 bytes was performed.)
See also
R_OSAL_IoDeviceOpen()

osal_device_handle_t *const  p_handle 
)

Open an IO device handle corresponding to p_id and set the handle to p_handle.

Parameters
[in]p_idTo set the value of target device name (a.k.a device id) acquired by R_OSAL_IoGetDeviceList().

Returns
e_osal_return_t
Return values

OSAL_RETURN_PARInvalid argument.
  • p_id is NULL.
  • p_handle is NULL.
OSAL_RETURN_MEMResource allocation failed.
OSAL_RETURN_DEVHW fault detected.
OSAL_RETURN_CONFWrong configuration value.
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_BUSYSpecified id is already handled. If this error occurs, p_handle will not be overwritten.
OSAL_RETURN_IDSpecified id is unknown.
OSAL_RETURN_STATENot initialized. (Before calling R_OSAL_Initialize().)
Note
After calling this function the OSAL Manager can access the target device via the device p_handle.
If this API returns an error code other than OSAL_RETURN_BUSY, p_handle will be overwritten with OSAL_DEVICE_HANDLE_INVALID.

◆ R_OSAL_IoGetAxiBusIdFromDeviceName()

e_osal_return_t R_OSAL_IoGetAxiBusIdFromDeviceName ( const char *  p_device_name,
osal_axi_bus_id_t *const  p_axi_id 
)

Returns the axi bus id for the given device. The id may refer to a axi bus domain and may be shared with other peripherals. The id can be used as index in the string list provided by R_OSAL_IoGetAxiBusNameList() to acquire a human readable name.

Parameters
[in]p_device_nameTo set the value of target device name (a.k.a device id).
[in,out]p_axi_idTo set the address of osal_axi_bus_id_t. The OSAL sets the identifier of bus name.
Returns
e_osal_return_t
Return values
OSAL_RETURN_OKAll arguments are valid, and the axi bus id for the given device name was got successfully.
OSAL_RETURN_PARInvalid argument.
  • p_device_name is NULL.
  • p_axi_id is NULL.
OSAL_RETURN_FAIL
  1. An error occurred in OS or related API.
  2. if the axi id for the given device name could not be acquired.
OSAL_RETURN_STATENot initialized. (Before calling R_OSAL_Initialize().)
OSAL_RETURN_UNKNOWN_NAMENo device was found related to p_device_name.
Note
This function have limited usage in drivers or middleware to limit string compares and improved performance. It may be used for debugging/tracing purposes in drivers/middlewares. Drivers/Middlewares are typically owner of the device handles and shall use R_OSAL_IoGetDeviceAxiBusId() instead.
This function will be deprecated and shall not be used.
See also
  • R_OSAL_IoGetAxiBusNameList()
  • R_OSAL_IoGetDeviceAxiBusId()

Get the string list of all axi bus names. The string list is a null terminated string with each entry delimited by character. e_osal_return_t
Return values

OSAL_RETURN_PARInvalid argument.

  • p_device_type_name is more than OSAL_DEVICE_TYPE_NAME_MAX_LENGTH.
  • p_num_of_byte is NULL.
  • p_buffer is NULL and buffer_size is not 0.
  • if provided buffer is too small.
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_STATENot initialized. (Before calling R_OSAL_Initialize().)
OSAL_RETURN_UNKNOWN_NAMENo device was found related to p_device_type_name.
OSAL_RETURN_OK_RQSTon success. Only the number of characters in p_buffer could be obtained.
  • p_buffer is NULL and buffer_size is 0.
Note
1. This function shall return an error if the given buffer is too small to hold all device ID strings which are must be returned.
  1. This function shall return the OSAL_RETURN_OK_RQST in case p_buffer==NULL && buffer_size==0 and update written_bytes to the required buffer size.
  2. This function shall modify the written_bytes output both on success and in the error case (if the provided buffer is too small).
  3. In case the provided buffer is too small this output variable shall be set to the required buffer size for the operation. Example: Requested device type is "abc" and it has 3 separate instances.
    Output of the function: Buffer with content: "abc_00\nabc_01\nabc_02\0".

Note

When accessing the same register from different threads or interrupts, ensure that there is no data race on the user side.
