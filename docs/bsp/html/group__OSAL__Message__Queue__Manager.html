◆ R_OSAL_MqCreate()

e_osal_return_t R_OSAL_MqCreate ( const st_osal_mq_config_t *const  p_config,
osal_mq_id_t  mq_Id,
osal_mq_handle_t *const  p_handle 
)

Create the message queue in the calling process as the mq_Id and configuration specified in p_config.

Parameters
[in]p_configThe address of st_osal_mq_config_t
[in]mq_IdThe value of message queue id.
[in,out]p_handleThe address of osal_mq_handle_t. The OSAL sets the handle.
Returns
e_osal_return_t
Return values
OSAL_RETURN_OKAll arguments are valid, and the message queue was created successfully.
OSAL_RETURN_PARInvalid argument.
  • p_config is NULL.
  • p_handle is NULL.
  • p_config->max_num_msg is 0.
  • p_config->msg_size is 0.
OSAL_RETURN_MEMResource allocation failed.
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_BUSYSpecified mq_id is already handled. If this error occurs, p_handle will not be overwritten.
OSAL_RETURN_IDSpecified mq_id is unknown.
OSAL_RETURN_STATENot initialized. (Before calling R_OSAL_Initialize().)
Note
In case of calling this API with a config initialized with R_OSAL_MqInitializeMqConfigSt(), OSAL_RETURN_PAR is returned. Before calling this API, the config value should be initialized with valid value. If this API returns an error code other than OSAL_RETURN_BUSY, p_handle will be overwritten with OSAL_MQ_HANDLE_INVALID.

◆ R_OSAL_MqReceiveForTimePeriod()

e_osal_return_t R_OSAL_MqReceiveForTimePeriod ( osal_mq_handle_t  handle,
osal_milli_sec_t  time_period,
void *  p_buffer,
size_t  buffer_size 
)

Receive a message from the message queue.
In case of the message queue is empty, wait for the time specified by time_period.

Parameters
[in]handleThe handle acquired at R_OSAL_MqCreate().
[in]time_periodSets the value of time to wait for the queue be able to receive message, in msec. (valid values are >= 0)
[in,out]p_bufferThe address of void* for the receives data. The OSAL sets the receive data.
[in]buffer_sizeThe p_buffer size by Byte. Should be equal msg_size in st_osal_mq_config_t.
Returns
e_osal_return_t
Return values
OSAL_RETURN_OKAll arguments are valid, and the message was received successfully from the message queue.
OSAL_RETURN_PARInvalid argument.
  • time_period is less than 0.
  • p_buffer is NULL.
  • buffer_size is not equal msg_size.
OSAL_RETURN_TIMECould not receive message within the specified time.
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_BUSYCould not receive message. This error occurs only interrupt context.
OSAL_RETURN_HANDLEInvalid handle.
  • handle was not acquired by R_OSAL_MqCreate().
  • handle was already deleted by R_OSAL_MqDelete().
OSAL_RETURN_STATENot initialized. (Before calling R_OSAL_Initialize().)
Note
The waiting time may be longer than specified due to the influence of other threads.

◆ R_OSAL_MqReceiveUntilTimeStamp()

e_osal_return_t R_OSAL_MqReceiveUntilTimeStamp ( osal_mq_handle_t  handle,
const st_osal_time_t *const  p_time_stamp,
void *  p_buffer,
size_t  buffer_size 
)

Receive a message from the message queue.
In case of the message queue is empty, wait until the time that is specified by the time stamp is reached.
The value of time stamp should be acquired by R_OSAL_ClockTimeGetTimeStamp().

Parameters
[in]handleThe handle acquired at R_OSAL_MqCreate().
[in]p_time_stampSet the pointer address of st_osal_time_t as timestamp that wait until the queue be able to receive message.
[in,out]p_bufferThe address of void* for the receive data. The OSAL sets the receive data.
[in]buffer_sizeThe p_buffer size by Byte. Should be equal msg_size in st_osal_mq_config_t.
Returns
e_osal_return_t
Return values
OSAL_RETURN_OKAll arguments are valid, and the message was received successfully from the message queue.
OSAL_RETURN_PARInvalid argument.
  • p_buffer is NULL.
  • p_time_stamp is NULL.
  • p_time_stamp->tv_sec is out of range. please see also st_osal_time_t.
  • p_time_stamp->tv_nsec is out of range. please see also st_osal_time_t.
  • buffer_size is not equal msg_size.
OSAL_RETURN_TIMECould not receive message within the specified time.
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_BUSYCould not receive message. This error occurs only interrupt context.
OSAL_RETURN_HANDLEInvalid handle.
  • handle was not acquired by R_OSAL_MqCreate().
  • handle was already deleted by R_OSAL_MqDelete().
OSAL_RETURN_STATENot initialized. (Before calling R_OSAL_Initialize().)
Note
The minimum unit of time is 1 milli second. Digits less than 1 milli second is rounded up to 1 milli second. If the time stamp is past time, use the current time stamp. The waiting time may be longer than specified due to the influence of other threads.

e_osal_return_t R_OSAL_MqReset ( osal_mq_handle_t  handle)

Delete all messages in the message queue.
After this call, the message queue is emptied and the thread waiting to send the message is unblocked.

Parameters
[in]handleThe handle acquired at R_OSAL_MqCreate().
Returns
e_osal_return_t
Return values
OSAL_RETURN_OKAll arguments are valid, and delete successfully all messages in the message queue.
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_HANDLEInvalid handle.
  • handle was not acquired by R_OSAL_MqCreate().
  • handle was already deleted by R_OSAL_MqDelete().
OSAL_RETURN_STATENot initialized. (Before calling R_OSAL_Initialize().)
Note
The message queue itself is not deleted. After calling this function, the handle is still valid , and the message queue can be used.
