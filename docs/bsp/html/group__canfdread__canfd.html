Read from the message buffer assigned to the given instance.

Read from the CAN-FD driver's receive buffer. The data will stored in the buffer given by Par1. The lenghth of the data to read must be a multiple of the the defined can message length <r_cio_can_Msg_t>

Parameters
[in]Addr- Channel number of the given CANFD instance
[in]Par1- Buffer address (destination)
[in]Par2- length of data to read. If 0 is specified, it is not an error. Also, the read data is returned as 0.
[in]Par3- optional parameter (unused)
Returns
>= 0 - length of the read data
< 0 - NG
