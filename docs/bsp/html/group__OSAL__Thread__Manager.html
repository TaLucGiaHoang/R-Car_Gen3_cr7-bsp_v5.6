◆ R_OSAL_ThreadCreate()

e_osal_return_t R_OSAL_ThreadCreate ( const st_osal_thread_config_t *const  p_config,
osal_thread_id_t  thread_id,
osal_thread_handle_t *const  p_handle 
)

Create a new thread in the calling process as the configuration specified in p_config.

Parameters
[in]p_configThe address of st_osal_thread_config_t.
[in]thread_idThe value of thread id.
[in,out]p_handleThe address of osal_thread_handle_t. The OSAL sets the handle.
Returns
e_osal_return_t
Return values
OSAL_RETURN_OKAll arguments are valid, and the thread was created successfully.
OSAL_RETURN_PARInvalid argument.
  • p_config is NULL.
  • p_handle is NULL.
  • p_config->priority is out of range. please see also e_osal_thread_priority_t.
  • p_config->stack_size is 0.
OSAL_RETURN_MEMResource allocation failed.
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_BUSYSpecified thread_id is already handled. If this error occurs, p_handle is not overwritten.
OSAL_RETURN_IDSpecified thread_id is unknown.
OSAL_RETURN_STATENot initialized. (Before calling R_OSAL_Initialize().)
Note
In case of calling this API with a config initialized with R_OSAL_ThreadInitializeThreadConfigSt(), OSAL_RETURN_PAR is returned. Before calling this API, the config value should be initialized with valid value.
If the func and userarg in p_config are NULL, successfully and return OSAL_RETURN_OK. OSAL does not provide the thread cancellation. Please prepare termination logic in the thread function.
If this API returns an error code other than OSAL_RETURN_BUSY, p_handle will be overwritten with OSAL_THREAD_HANDLE_INVALID.
See also
  • st_osal_thread_config_t
  • e_osal_thread_priority_t

◆ R_OSAL_ThreadSleepUntilTimeStamp()

e_osal_return_t R_OSAL_ThreadSleepUntilTimeStamp ( const st_osal_time_t *const  p_time_stamp)

Suspend the execution of the current thread until an absolute time value specified by p_time_stamp is reached.
The value of time stamp should be acquired by R_OSAL_ClockTimeGetTimeStamp().

Parameters
[in]p_time_stampThe pointer addresses of st_osal_time_t that holds the timestamp on which the thread will continue.
Returns
e_osal_return_t
Return values
OSAL_RETURN_OKAll arguments are valid, and the thread was suspended successfully.
OSAL_RETURN_PARInvalid argument.
  • p_time_stamp is NULL.
OSAL_RETURN_FAILAn error occurred in OS or related API.
OSAL_RETURN_STATENot initialized. (Before calling R_OSAL_Initialize().)
Note
The minimum unit of time is 1 milli second. Digits less than 1 milli second is rounded up to 1 milli second. If the time stamp is past time, use the current time stamp.
The waiting time may be longer than specified due to the influence of other threads.
